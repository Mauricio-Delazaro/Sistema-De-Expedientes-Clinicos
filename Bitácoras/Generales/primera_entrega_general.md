## Resumen de actividades por periodo

- **Semana 1:**
	- Se identificaron, redactaron y organizaron los requisitos funcionales del sistema, definiendo las funcionalidades principales que deberá proporcionar a los distintos actores.
	- Se identificaron y redactaron los requisitos no funcionales del sistema, estableciendo las características de calidad y restricciones que deberá cumplir el sistema.

## Avances respecto al estado inicial

Antes de la primera entrega no existía una especificación formal de requisitos para el módulo de expedientes clínicos. Durante esta entrega se lograron los siguientes avances principales:

- **Definición de requisitos funcionales:**
	- Se estableció un conjunto inicial de requisitos funcionales que describen qué debe hacer el sistema para soportar a los distintos actores, proporcionando una base clara para el modelado posterior de casos de uso, clases y arquitectura.

- **Definición de requisitos no funcionales:**
	- Se documentaron requisitos no funcionales que describen atributos de calidad y restricciones del sistema (por ejemplo, desempeño, seguridad, confiabilidad), creando el punto de partida para la posterior especialización de requisitos como conservación de expedientes, control de acceso, integridad y auditoría.

En conjunto, la primera entrega sentó las bases conceptuales del sistema, sobre las que se profundizó en entregas posteriores (detalle de RNF específicos, arquitectura, modelo de datos y reglas de negocio).
